# Автоматизация SMS для CRM (Notion + Claude + SMS.ru)

Два бесплатных сценария на GitHub Actions, тексты SMS — фиксированные шаблоны (без ИИ-генерации):
- `feedback-request.js` — раз в час проверяет заказы со статусом «Выполнено» и шлёт SMS с просьбой оставить отзыв.
- `daily-reminder.js` — раз в день шлёт 5 клиентам напоминание о себе.

Тексты сообщений лежат в `scripts/templates.js` — правьте прямо там.

Стоимость: только сами SMS в SMS.ru. GitHub Actions и Notion API — бесплатно.

## Шаг 1. Создайте интеграцию Notion

1. Откройте https://www.notion.so/my-integrations → **New integration**.
2. Дайте любое имя, привяжите к своему workspace, сохраните.
3. Скопируйте **Internal Integration Secret** — это и есть `NOTION_TOKEN`.
4. Откройте в Notion базу «Заказы» → `...` (три точки вверху) → **Connections** → добавьте вашу интеграцию. Повторите то же самое для базы «Клиенты».
   (Без этого шага API будет отвечать «objects not found», даже если токен верный.)

## Шаг 2. Получите API ID в SMS.ru

Личный кабинет sms.ru → раздел «Разработчикам» → скопируйте `api_id`. Это `SMS_RU_API_ID`.
Пополните баланс на небольшую сумму для тестов.

## Шаг 3. Загрузите этот проект в свой репозиторий на GitHub

```bash
cd crm-automation
git init
git add .
git commit -m "SMS automation"
git branch -M main
git remote add origin https://github.com/<ваш-логин>/<репозиторий>.git
git push -u origin main
```

Репозиторий может быть приватным — это ничего не меняет для GitHub Actions.

## Шаг 4. Добавьте секреты в репозиторий

В репозитории: **Settings → Secrets and variables → Actions → New repository secret**. Добавьте два секрета:
- `NOTION_TOKEN`
- `SMS_RU_API_ID`

Никогда не вписывайте эти значения прямо в код — только через секреты.

## Шаг 5. Проверьте вручную

Вкладка **Actions** в репозитории → выберите один из двух workflow → **Run workflow** — запустит вручную, не дожидаясь расписания. Проверьте логи запуска — там видно, что скрипт нашёл и отправил (или почему нет).

## Если что-то не совпадает в вашей базе

Все названия полей и ID баз собраны в одном месте — `scripts/config.js`. Если вы переименуете свойство в Notion, поправьте его название и там.

## Тексты сообщений

Оба текста — в `scripts/templates.js`:
- `feedbackTemplate` — уже заполнен вашим текстом для отзыва.
- `reminderTemplate` — пока черновик-заглушка (одна фраза-напоминание). Пришлите свой вариант(ы) текста — можно 2-3 штуки, тогда скрипт будет выбирать случайный, чтобы рассылка не выглядела как спам с одинаковым текстом.

## Частота и расписание

- `feedback-request.yml` — запускается каждый час (`0 * * * *`). Можно сделать чаще, например каждые 15 минут (`*/15 * * * *`), учитывайте лимиты бесплатного плана GitHub Actions (2000 минут/мес для приватных репозиториев — для такой лёгкой задачи с большим запасом).
- `daily-reminder.yml` — раз в день в 10:00 по Москве.
